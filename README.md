# 🌀 Gravity Shift

> *"점프만으로는 닿을 수 없는 곳이 있다. 중력이 곧 길이다."*

3D 퍼즐 점프맵. 구체를 굴리면서 **위/아래 중력 반전**으로 천장도 바닥처럼 활용해 골에 도달합니다.

## 실행

빌드 없이 정적 호스팅만 있으면 동작합니다 (ES 모듈 + Three.js CDN).

```bash
python3 -m http.server 8080
# 브라우저에서 http://localhost:8080
```

또는 GitHub Pages / Vercel / itch.io 등 어디든 정적 배포.

## 조작

| 입력 | 동작 |
|---|---|
| `W A S D` | 현재 중력의 수평면 기준 이동 |
| `SPACE` | 점프 (현재 중력 반대 방향) |
| `Q` | 중력 반전 (↓ ↔ ↑ 토글) |
| 마우스 | 카메라 회전 (캔버스 클릭으로 잠금) |
| `R` | 리스폰 |
| `ESC` | 일시정지 |

## 스테이지

| ID | 이름 | 학습 포인트 |
|---|---|---|
| 1-1 | 첫 걸음 | 기본 이동 / 점프 |
| 1-2 | 위가 아래다 | 첫 반전 — 천장 가로질러 벽 너머로 |
| 1-3 | 더 높이 | 천장에서 떨어지며 못 닿는 발판 착지 |
| 2-1 | 천장 다리 | 점프 불가 갭을 천장 다리로 건너기 |
| 2-2 | 양면 통로 | 바닥/천장 장애물이 번갈아, 반전으로 위빙 |
| 3-1 | 반전 3회 | 제한된 반전 횟수로 정확한 풀이 |

## 구조

```
src/
├── main.js                  # 게임 부트스트랩 / 메인 루프
├── input.js                 # 키보드 + 포인터 락 입력
├── game/
│   ├── Player.js            # 구체 캐릭터 + 커스텀 물리 (sphere vs AABB)
│   ├── GravityController.js # ±Y 중력 토글 + 전환 횟수 제한
│   ├── CameraRig.js         # 중력 정렬 3인칭 카메라
│   └── LevelLoader.js       # JSON 스타일 레벨 → 씬
├── levels/levels.js         # 스테이지 데이터
└── ui/
    ├── HUD.js               # 게이지/타이머/방향 표시기
    └── style.css

public/models/sphere.glb     # (선택) 사용자 구체 모델을 여기 두면 자동 사용
```

GLB 모델을 `public/models/sphere.glb`에 두면 자동으로 로드됩니다. 없으면 절차적 구체 메시로 동작합니다.

## 메모

- 물리는 외부 라이브러리 없이 커스텀 구현 (구체 vs AABB). 중력 벡터를 동적으로 바꾸는 게 핵심이라 직접 통제하는 편이 쉬워서.
- 카메라의 up 벡터는 -gravity. 안티포달 플립(180°) 시 카메라의 right 축을 기준으로 돌려서 어지럽지 않도록 처리.
- 베스트 타임은 `localStorage`에 저장.

GDD 전체 사양(레벨 에디터, 음향, 모바일 등)은 이번 프로토타입 범위 밖.
